# Google AI Search MCP 使用指南

## ⚠️⚠️⚠️ 最高优先级规则 ⚠️⚠️⚠️

**在开始任何代码修改之前，必须先执行搜索！**

当用户报告 bug、问题、错误时：
1. **第一步**：使用 `mcp_google_ai_search_google_ai_search` 搜索问题的常见原因和解决方案
2. **第二步**：阅读搜索结果，了解业界已知的解决方案
3. **第三步**：再开始读代码、加日志、修改代码

**禁止行为**：直接开始读代码调试，跳过搜索步骤

## 🌟🌟🌟 Google AI Search - 全能通用搜索工具（最重要！）🌟🌟🌟

> **核心理念：有任何不确定的事情，先搜一下确认！宁可多搜也不要凭记忆猜测！**

**这是你的首选搜索工具，适用于几乎所有需要获取外部信息的场景。**

不要犹豫，不要等待用户明确要求 —— 遇到任何不确定的问题，主动搜索确认！搜索成本很低，但错误的信息代价很高。

**工具**：`mcp_google_ai_search_google_ai_search`

**参数**：
- `query`（必填）：向 Google AI 提问的自然语言问题
  - ⚠️ 重要：使用完整的自然语言句子提问，而非关键词堆砌！
- `language`（可选）：搜索结果语言，默认 `zh-CN`
  - `zh-CN` - 中文结果
  - `en-US` - 英文结果
  - `ja-JP` / `ko-KR` / `de-DE` / `fr-FR` - 其他语言

## 搜索策略指南

### ✅ 应该搜索（强烈建议）

- **实时/时效性信息**: 最新版本号、近期发布、当前价格、最新动态
- **具体产品/服务细节**: 特定 API 的最新用法、某产品的具体配置参数
- **行业最新实践**: 2024/2025 年的最佳实践、新兴技术趋势
- **争议性/无定论问题**: 不同方案的优劣对比、社区讨论热点
- **小众/冷门知识**: 特定框架的边缘用法、罕见错误的解决方案
- **任何你不确定的事情**: 与其猜测，不如花几秒搜索确认

### ❌ 可以不搜索（但搜了也没坏处）

- **基础概念**: 什么是 REST API、JavaScript 闭包原理
- **稳定的语法/用法**: Python 列表操作、SQL 基本语法
- **通用设计模式**: 单例模式、观察者模式的基本实现
- **AI 训练数据内的知识**: 经典算法、成熟框架的常规用法
